# Lesson 1.2: The Anatomy of a Perfect Prompt

If you write prompts like you're talking to a search engine or chatting with a friend, you will get inconsistent, unpredictable answers. To write a perfect prompt that succeeds on the first try, you need to structure your request into four distinct pillars.

---

## 🏛️ The 4-Part Prompt Framework

Every robust prompt should isolate the following four elements clearly:

1. **Role / Persona (Who is the model?):** Sets the expert boundaries, tone, and domain limits.
2. **Context (What is the background?):** Providing the raw materials, documents, constraints, or datasets to analyze.
3. **Instructions / Task (What must it do?):** Precise actions using imperative verbs (e.g., "Analyze," "Reformat," "Check").
4. **Constraints & Output Formatting (How must it return it?):** Specific formats like JSON, bulleted Markdown, or restricted lengths.
