# Lesson 1.2: The Anatomy of a Perfect Prompt

If you write prompts like you're talking to a search engine or chatting with a friend, you will get inconsistent, unpredictable answers. To write a perfect prompt that succeeds on the first try, you need to structure your request into four distinct pillars.

---

## 🏛️ The 4-Part Prompt Framework

Every robust prompt should isolate the following four elements clearly:

1. **Role / Persona (Who is the model?):** Sets the expert boundaries, tone, and domain limits.
2. **Context (What is the background?):** Providing the raw materials, documents, constraints, or datasets to analyze.
3. **Instructions / Task (What must it do?):** Precise actions using imperative verbs (e.g., "Analyze," "Reformat," "Check").
4. **Constraints & Output Formatting (How must it return it?):** Specific formats like JSON, bulleted Markdown, or restricted lengths.

---

## 🎛️ Prompt Template Blueprint

In a programmatic pipeline, your prompt template is stored as a string with placeholders (variables) that you inject at runtime:

```text
ROLE: You are an expert {persona}.

CONTEXT DATA:
---
{input_data}
---

TASK:
{task_description}

CONSTRAINTS / FORMATTING:
- Restrict output format exactly to {output_format}.
- Do not add conversational intro or outro text.
```

### ⚙️ Runtime Variable Injection Example
If you assign the following Python variables:
* `persona` = `"Technical Project Manager"`
* `input_data` = `"Sarah will update the database schema by Friday. John will coordinate with marketing."`
* `task_description` = `"Extract action items, assignees, and deadlines."`
* `output_format` = `"A numbered markdown list"`

The final prompt automatically compiles into a clean, structured command ready for the LLM API.

## 👥 How the 4-part framework applies across roles

See how the framework translates into specific professional fields:

<Tabs>
  <Tab label="📣 Marketer">
    * **Role:** Direct-response growth marketer.
    * **Context:** Product benefits sheet and target customer demographic profiles.
    * **Task:** Craft 3 variants of email subject lines.
    * **Constraints:** Must be under 60 characters, with high open rate appeal.
  </Tab>
  <Tab label="🔬 Researcher">
    * **Role:** Scientific research database analyst.
    * **Context:** Text snippet from a journal article PDF.
    * **Task:** Extract methodologies and sample sizes.
    * **Constraints:** Output as a Markdown table. Do not make assumptions.
  </Tab>
  <Tab label="📋 Project Manager">
    * **Role:** Technical Project Manager.
    * **Context:** Raw meeting transcripts containing random side talks.
    * **Task:** Filter conversations to build a task checklist.
    * **Constraints:** Map tasks as key/value pairs in JSON format: `{"task": "...", "assignee": "..."}`.
  </Tab>
  <Tab label="💼 Analyst">
    * **Role:** IT operations auditor.
    * **Context:** Server log reports.
    * **Task:** Identify the IP addresses with the highest error rates.
    * **Constraints:** Restrict response to a bulleted list of IPs and their incident frequencies.
  </Tab>
</Tabs>
